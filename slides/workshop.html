Limitations & Best Practices

Claude writes the code.
You make the scientific decisions.

Be aware
Code can "work" and still be wrong
The model converges, the output looks plausible — but missing confounders or violated assumptions are silent failures no test suite catches
Training data has a cutoff
May suggest httr instead of httr2, plyr instead of dplyr, or outdated Bioconductor APIs. CLAUDE.md and MCP servers keep it current
It doesn't see your data by default
May reference columns that don't exist or apply transformations that don't fit your types — always direct it to read the actual files
Best practices
Use CLAUDE.md to enforce conventions
Specify: native pipe |>, tidyverse style, set.seed() in every script, preferred packages and versions
Review like you'd review a colleague's code
Check package versions, data assumptions, statistical choices, and pipe syntax. Same scrutiny, different author
Use freely for coding, cautiously for analysis
Scaffolding, syntax, boilerplate, debugging — reliable. Model selection, transformation decisions, interpretation — your judgment, always
Your analytical judgment is the thing that makes research trustworthy — no tool changes that
